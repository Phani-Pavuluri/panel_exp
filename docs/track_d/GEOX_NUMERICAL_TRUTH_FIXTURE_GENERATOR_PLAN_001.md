# GEOX_NUMERICAL_TRUTH_FIXTURE_GENERATOR_PLAN_001

## Metadata / Purpose
Plan for a future deterministic local generator supporting the 12 numerical-truth cases; no files or datasets are generated by this artifact.

## Artifact layout and case mapping
Future artifacts live under `tests/fixtures/geox_numerical_truth/manifest.json/<case_id>/` with `input_panel.csv`, `truth.json`, `expected_readout.json`, optional `expected_packet.json`, and `replay.json`. The manifest indexes all 12 IDs, required versus optional files, hashes, versions, and certification status.

## Generator behavior
The future generator uses no network or external data, stable random and assignment seeds, stable panel grain/geo/time/KPI/estimand, reproducible treatment/control construction, known effects, expected uncertainty, exact blockers/warnings, content fingerprints, idempotent regeneration, validate-before-write, safe overwrite only for matching versions, and rollback by removing only generated case directories.

## Future API plan
Planned but not implemented: `build_geox_numerical_truth_fixture_artifacts(case_id)`, `build_all_geox_numerical_truth_fixture_artifacts()`, and `validate_generated_geox_truth_fixture_artifacts(root)`. Failures include missing case, schema mismatch, unstable hash, panel mismatch, truth mismatch, blocker mismatch, and unsafe production status.

## Twelve planned cases
`geox_truth_scm_candidate_clean_001`, `geox_truth_did_candidate_warning_001`, `geox_truth_infeasible_preperiod_001`, `geox_truth_weak_matchability_001`, `geox_truth_unsupported_inference_001`, `geox_truth_tbrridge_diagnostic_only_001`, `geox_truth_bayesian_tbr_research_only_001`, `geox_truth_stale_incompatible_evidence_001`, `geox_truth_conflicting_evidence_001`, `geox_truth_multicell_shared_control_block_001`, `geox_truth_calibration_incompatible_001`, and `geox_truth_safe_blocked_readout_001` each receive input, truth, expected readout/packet, manifest, replay, and tolerance expectations.

## Validation plan
Require twelve directories and manifest entries; required files; truth-contract validation and round trip; panel grain/time-window and unit consistency; blocked/diagnostic/research non-production semantics; estimate/uncertainty tolerance; exact warning/blocker/calibration/handoff states; stable hashes across regeneration.

## Readout alignment / MIP handoff
Successful cases feed governed readout candidates, warnings feed accepted-with-warning, blocked cases packets, diagnostic/research cases constrained artifacts. Calibration-incompatible cases cannot become CalibrationSignal; multicell blocks remain non-production. MIP consumes artifacts but does not certify GeoX truth.

## D6 Gate 1 alignment
The plan supports but does not complete D6 Gate 1. Remaining requirements: producer/consumer versions, compatibility matrix, ownership, required/optional fields, failure semantics, release/rollback order, limitations, owners, authorization flags, and migration/deprecation rules.

## Non-goals / Authorization boundary
No fixture files, generator code, readout runtime, entrypoint, estimator, assignment, MIP/MMM change, export, reporting, decisioning, agent, real-data pilot, or production authorization.
